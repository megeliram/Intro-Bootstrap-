Mobile first and controlling placement

Controlling Placement

Offset

  • Flows in order
  • Leaves specified columns blank
  • Offset will instruct Bootstrap to skip the specified number of columns before placing content. With offset, the skipped columns will be left blank.
    Use to center things
  • Example: Offset 2 => Push 2 columns to the right and leaves the last 2 columns blank
Push
  • Move an item to the RIGHT
  • Allows the item to be specified EARLIER than flow
  • Push will instruct Bootstrap to move content to the right a specified number of columns. With push, the columns that were left blank can be used by pulling content to the left.
Pull
  • Move an item to the LEFT
  • Allows the item to be specified LATER than flow
  • Pull will instruct Bootstrap to move content to the left a specified number of columns. With pull, the columns that were left blank can be used by pushing content to the right.
Note* Columns are not left blank with Pull/Push. Only Offset leaves columns vacated.


Example: col-xs-push-6
col - abbr. for screen size - option - # of columns
This will move content to the right 6 columns on extra small screens

Mobile First

It is best practice to design for mobile - or smallest - device first. It's easier to scale content to a larger size than it is to shrink content.

Bootstrap is built with mobile-first in mind
