Module 1: Page Layout

Page Layout: Three common starting classes

Container

The container class places content inside of a horizontal container. The size of the container will vary based on screensize. The container will stay the same width for that particular screensize. This helps developers size content appropriately for different devices.

Container-fluid

The container-fluid class places content inside of a container that will always be the width of the screen. This is helpful for scenarios where the content to be displayed must use the entire browser window.

Jumbotron

The jumbotron class is a common class for displaying titles for different sections of a website, or the landing page. The jumbotron typically has a highlighted background and an increased fontsize.

Grids

  • Never use table structures when developing for page layout. It degrades web performance of page layout
  • Use BS grid system instead

  • Four screen sizes in Grids
  • Always 12 columns
  • Smaller sizes set the default for larger sizes
  • col-sm - sets size for small, medium, and large screens unless overriden by a setting to one of the larger sizes
  • there is a 30px "gutter" or empty space between each column (15px) on each side of column
  • the gutter on the outside of container varies based on screen size
  • extra small devices always use the entire width of screen

    • large (1200px) - desktops
    • medium (992-1200px) - large tablets and laptops
    • small (768-991px) - tablets
    • extra small (less than 768px) - phones

    Additional Notes

    • Flow container will take the entire screen size, size is going to vary
    • If you are using a fixed container
